Einige von euch haben bei Übung 04 Probleme, wenn das Pixel-Array zu groß wird (also zu viele Pixel enthält). Die einfachste Lösung, um euren Code zu testen, ist den Input kleiner zu machen, z.B. mit https://www.iloveimg.com/resize-image. Auch können wir statt `int` den `unsigned char` Datentyp für das Pixel-Array verwenden. Dieser kann nur Werte von 0 bis 255 abbilden und verbraucht deshalb weniger Speicher als ein regulärer `int`.
Falls ihr das Problem habt, muss euer Code nicht falsch sein. In unterschiedlichen Entwicklungsumgebungen kann es unterschiedliche Oberlimits geben, ab denen das Pixel-Array zu groß wird. Bitte testet euren Code mit kleineren Inputs und erwähnt in einem Kommentar, falls ihr auf diese Problematik gestoßen seid.
Hintergrund:
Das Problem entsteht, wenn versucht wird zu viel Speicher auf dem Stack (dem “Standardspeicher” in C) zu allozieren. Dieses Limit kann je Compiler, Maschine und Betriebssystem unterschiedlich sein. Die fortgeschrittene Lösung wäre das Problem zu umgehen, indem wir mit `malloc` das Array auf dem Heap statt auf dem Stack allozieren. Das werden wir allerdings erst später in der Vorlesung behandeln.
